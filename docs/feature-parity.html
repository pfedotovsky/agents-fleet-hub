Agents Hub vs. CloudCLI Web App — Feature Parity

Agents Hub (this repo, fleet-hub/) vs. CloudCLI v1.36.1 web UI. First compared 2026-07-11; status updated 2026-07-12. Canonical source: feature-parity.md (this HTML is a regenerated view). Planned work lives in docs/backlog.md.

Agents Hub is a multi-host aggregator over per-VM CloudCLI APIs; CloudCLI's own UI is a single-host full-featured client. Many CloudCLI features are server APIs Agents Hub can adopt with frontend-only work; a few (plugins, Electron) are out of scope for a static hub SPA.

Agents Hub advantages (no CloudCLI equivalent)

  • Multi-host aggregation: one pane of glass over N VMs, cross-host activity feed, per-host colors and status dots.
  • Cross-host full-text search (⌘K) fanning out to every online host.
  • Plan-mode toggle (Shift+Tab) with a docked plan drawer; AskUserQuestion answer cards.
  • Sessions started elsewhere (terminal, host UI) flow in via session_upserted + fallback polling.
  • Permission grants persisted per host:project and written through to .claude/settings.local.json.
  • Offline/hibernating VM handling with retained last-known state and setup hints.
  • One-click new session, drag-resizable sidebar, active-session-first ordering.
