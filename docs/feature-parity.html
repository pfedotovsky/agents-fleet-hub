Agents Hub vs. CloudCLI Web App — Feature Parity

Agents Hub (this repo, fleet-hub/) vs. CloudCLI v1.36.1 web UI. Generated 2026-07-11.

Agents Hub is a multi-host aggregator over per-VM CloudCLI APIs; CloudCLI's own UI is a single-host full-featured client. Many CloudCLI features are server APIs Agents Hub can adopt with frontend-only work; a few (plugins, Electron) are out of scope for a static hub SPA.

✅ full⚠️ partial❌ missing➕ Agents Hub advantage

Agents Hub advantages (no CloudCLI equivalent)

  • Multi-host aggregation: one pane of glass over N VMs, cross-host activity feed, per-host colors and status dots.
  • Sessions started elsewhere (terminal, host UI) flow in via session_upserted + fallback polling.
  • Permission grants persisted per host:project and written through to .claude/settings.local.json.
  • Offline/hibernating VM handling with retained last-known state and setup hints.
  • One-click new session, drag-resizable sidebar, active-session-first ordering.

P2 — High value, more work

  • ☐ Integrated terminal — xterm.js against the host's shell WebSocket (PTY, resize, agent CLI or plain shell). Biggest single UX gap.
  • ☐ URL routing — deep-linkable host/project/session views, back-button support; prerequisite for PWA.
  • ☐ Mobile/responsive layout + PWA — collapsible sidebar, manifest + service worker.
  • ☐ Web Push notifications — subscribe to each host's VAPID push so alerts arrive with the tab closed.
  • ☐ Token usage / context-window display — where supportsTokenUsage (claude/codex/opencode).
  • ☐ Slash commands — fetch /api/commands per project, composer autocomplete.
  • ☐ Session rename — investigate mechanism (plugin-based?); implement if a stable API exists.

P3 — Nice to have

  • ☐ MCP server management UI — per-provider + global over /:provider/mcp/servers.
  • ☐ Light theme / appearance settings.
  • ☐ KaTeX + Mermaid rendering in messages.
  • ☐ File operations — create/rename/delete (needs shell or new host support).
  • ☐ Voice input — mic → host /api/voice (needs host-side OpenAI key).
  • ☐ Skills management UI.
  • ☐ TaskMaster integration — tasks + PRD tooling per project.
  • ☐ Conversation compaction control, if exposed over WS.

Explicitly out of scope

Plugin system/marketplace, browser-use automation, Electron build, API-key/credential admin, onboarding/git-identity setup — host-local features or ones requiring a backend the hub deliberately doesn't have.
